## 4. 页面功能测试范围
按主流程、边界值、异常态、回归项拆分如下。

### 4.1 主流程范围
1. 进入商品对比页，页面完成基础渲染。
2. 从商品添加入口添加 2 至 4 个不同商品进入对比列表。
3. 对比列表展示商品基础信息，包括名称、图片、价格、库存、核心属性或规格字段。
4. 参数对比区域展示各商品属性，并可区分相同项与差异项。
5. 价格与库存区域展示对应字段，字段完整时显示真实值。
6. 点击商品详情入口，跳转到对应商品详情页或详情路由。
7. 删除其中一个商品后，对比列表与参数区域刷新为最新状态。
8. 删除后页面恢复可继续添加商品的能力。

### 4.2 边界值范围
1. 重复添加同一商品。
2. 添加商品达到数量上限。
3. 在已达上限后继续尝试添加商品。
4. 商品价格缺失。
5. 商品库存为 0。
6. 商品库存字段缺失或异常值。
7. 参数字段过长导致换行或布局挤压。
8. 参数字段部分缺失。
9. 接口返回空数组但页面仍处于可操作状态。

### 4.3 异常态范围
1. 页面首次加载中。
2. 页面首次加载失败。
3. 页面首次加载超时。
4. 添加商品请求失败。
5. 移除商品后刷新请求失败。
6. 对比数据接口返回部分商品缺字段。
7. 对比数据接口返回部分商品失败。
8. 参数区局部渲染失败。
9. 价格库存区域局部渲染失败。
10. 空数据返回。
11. 重试后恢复成功。

### 4.4 回归项范围
1. 重复添加被拦截时，已有对比数据不被破坏。
2. 达上限时，已有商品顺序、内容、差异高亮状态不被破坏。
3. 删除商品后，剩余商品顺序与数量正确。
4. 错误提示、空态、加载态切换后，不影响详情跳转入口可用性判断。
5. 异常重试成功后，页面状态恢复到正常展示，不残留错误文案或脏数据。

## 5. 页面模块测试拆分

### 5.1 商品添加入口组件
覆盖点：

1. 可添加商品进入对比。
2. 重复商品添加拦截。
3. 达到数量上限后的提示、按钮状态或入口状态。
4. 搜索/选择结果为空时的表现。
5. 添加请求失败、超时后的提示与恢复。
6. 添加成功后对比列表刷新联动。

### 5.2 对比列表组件
覆盖点：

1. 已添加商品卡片数量与顺序展示正确。
2. 商品基础信息完整展示。
3. 删除单个商品后列表刷新。
4. 删除后占位态更新。
5. 删除后可继续添加状态恢复。
6. 列表为空时空态展示。
7. 超限提示出现时，列表数据不变。

### 5.3 参数对比区域组件
覆盖点：

1. 相同参数正常展示。
2. 差异参数高亮展示。
3. 非差异参数不误高亮。
4. 参数缺失时展示容错。
5. 长字段折行或截断策略稳定。
6. 删除商品后参数列数同步刷新。
7. 空对比数据时参数区空态或隐藏逻辑正确。

### 5.4 价格与库存展示组件
覆盖点：

1. 价格正常值展示。
2. 价格缺失时展示文案或占位样式。
3. 库存大于 0 的正常展示。
4. 库存为 0 的缺货或无库存展示。
5. 库存缺失或异常值的容错展示。
6. 局部加载中状态。
7. 局部失败状态。

### 5.5 商品对比页面容器
覆盖点：

1. 首屏加载态。
2. 空数据态。
3. 整体错误态。
4. 重试逻辑。
5. 页面级数据组装后对子组件的传递一致性。
6. 详情跳转事件透传与字段完整性。

## 6. 接口联调范围
按接口行为和返回场景梳理如下。若实际仓库中接口已拆分为多个 API，以实际命名映射即可。

### 6.1 对比数据查询接口
目标：验证商品基础信息、价格、库存、属性字段完整性与容错。

覆盖场景：

1. 成功返回完整商品列表。
2. 成功返回空数组。
3. 成功返回部分字段缺失。
4. 成功返回部分商品异常数据。
5. 返回 500 或业务错误码。
6. 返回超时或网络异常。

关键断言：

1. 页面是否正确渲染商品基础信息。
2. 差异高亮是否与返回字段一致。
3. 缺失字段是否走容错展示。
4. 空数组是否进入空态。
5. 失败与超时是否进入错误态，并支持重试。

### 6.2 添加对比商品接口
目标：验证重复商品拦截、数量上限与前端处理一致性。

覆盖场景：

1. 添加成功并返回最新对比列表。
2. 添加重复商品，前端本地拦截或接口返回重复标识。
3. 添加超上限商品，前端本地拦截或接口返回上限错误。
4. 添加失败。
5. 添加超时。

关键断言：

1. 成功后列表数量增加。
2. 重复商品不进入列表。
3. 超上限时已有数据不受影响。
4. 错误提示与按钮状态恢复正确。

### 6.3 移除对比商品并刷新接口
目标：确认删除后最新列表返回并驱动页面重绘。

覆盖场景：

1. 删除成功并返回最新列表。
2. 删除成功后列表为空。
3. 删除失败。
4. 删除后刷新失败或超时。

关键断言：

1. 剩余商品顺序正确。
2. 参数区和价格库存区同步刷新。
3. 空列表时展示空态或占位。
4. 可添加状态恢复。

### 6.4 详情跳转依赖字段
目标：校验商品详情跳转依赖字段完整有效。

覆盖场景：

1. 返回商品 ID 可供路由跳转。
2. 返回详情链接可直接打开。
3. 返回路由参数对象或 slug。
4. 缺失跳转字段。

关键断言：

1. 存在有效字段时可正确跳转。
2. 字段缺失时页面需有禁用、降级或无动作保护。

## 7. 页面状态范围

| 状态类型 | 触发条件 | 预期页面表现 | 备注 |
| --- | --- | --- | --- |
| 加载中 | 首次进入页面或重试中 | 展示页面级 loading，占位或骨架 | 具体样式待确认 |
| 空数据 | 对比接口成功返回空数组 | 展示空态、可引导继续添加商品 | 引导文案待确认 |
| 整体异常 | 页面主接口失败或超时 | 展示错误提示、重试入口 | 错误文案待确认 |
| 局部异常 | 价格库存或参数区域局部失败 | 局部区域降级，不应导致整页崩溃 | 失败口径待确认 |
| 部分数据缺失 | 某商品缺价格/库存/属性 | 缺失字段走容错展示，不显示脏值 | 容错文案待确认 |
| 删除后空态 | 移除至 0 个商品 | 列表、参数区、价格库存区同步进入空态或占位态 | 表现口径待确认 |
| 达上限 | 已添加商品达到限制 | 添加入口禁用或提示限制 | 上限值待确认 |
| 重复添加 | 添加已在列表中的商品 | 给出提示，列表不新增 | 判重依据待确认 |

## 9. 数据分类与用例组织建议
后续测试数据建议至少按以下类别组织，便于复用 mock 与联调样例：

1. 正常商品：字段完整，可参与标准对比。
2. 重复商品：与已添加商品具备相同判定主键。
3. 超上限商品：用于验证限制提示与数据保护。
4. 缺字段商品：缺价格、缺库存、缺属性、缺详情字段。
5. 无库存商品：库存为 0。
6. 差异商品组合：用于验证高亮。
7. 无差异商品组合：用于验证不误高亮。
8. 异常接口样例：500、超时、空数组、部分失败。

建议每组数据至少显式包含以下字段：

1. 商品 ID
2. 名称
3. 图片
4. 价格
5. 库存
6. 规格属性
7. 差异标记或可推导差异字段
8. 详情链接或路由字段
9. 接口状态码
10. 错误信息

## 10. 最终验收口径

### 10.1 必须覆盖的验收项
1. 已形成覆盖页面功能、接口联调和页面状态异常的测试范围清单。
2. 已建立功能点与页面模块、组件、接口返回场景的映射关系。
3. 已按主流程、边界值、异常态、回归项分类测试点。
4. 已明确商品添加、重复校验、数量上限、移除刷新、差异高亮、价格库存展示、空加载异常状态、详情跳转均在后续测试范围内。
5. 已明确对比数据查询、添加商品、移除刷新、详情跳转依赖字段等接口联调范围。
6. 已明确记录测试框架、Mock 方式、联调环境、页面选择器等基础能力需核对内容。
7. 所有影响测试结论的缺失项均已单独记录，且未自行补充业务规则。

### 10.2 后续自动化用例通过标准
进入下一步骤编写和执行测试时，以下场景应视为核心通过标准：

1. 至少覆盖 1 条页面主流程成功链路。
2. 至少覆盖重复添加与数量上限 2 类关键边界值。
3. 至少覆盖空数据、整体失败、超时重试 3 类关键状态场景。
4. 至少覆盖差异高亮、价格缺失、库存为 0 3 类关键展示场景。
5. 至少覆盖删除刷新与详情跳转 2 条回归链路。

### 10.3 阻塞判定规则
以下信息若在后续实现前仍未确认，将直接影响测试结论可信度，应标记为阻塞项：

1. 商品对比数量上限具体值。
2. 重复商品判定依据。
3. 差异高亮判定规则。
4. 价格或库存缺失时的展示文案。
5. 局部渲染失败的页面表现口径。
6. 详情跳转依赖字段来源。
7. 当前项目实际测试框架、Mock 方案、联调环境入口、可复用选择器规范。

| 局部渲染失败表现口径 | 参数区、价格库存区 | 无法界定是降级为空、错误提示还是隐藏模块 |
| 详情跳转依赖字段来源 | 商品卡片、跳转断言 | 无法稳定 mock 路由或链接字段 |

| 联调环境入口 | 联调用例执行 | 无法执行真实联调验证 |
| 页面选择器规范 | UI 自动化稳定性 | 无法编写稳健查询与交互断言 |

## 12. 后续步骤建议
1. 先在仓库中核实测试框架、Mock 能力、compare 页面实现位置和接口定义文件。
2. 基于本清单补充测试矩阵，沉淀为可执行测试用例文件或验收 checklist。
3. 对阻塞项逐条向产品、前端实现或接口负责人确认，避免自动化用例建立在错误规则上。
4. 再进入页面级测试代码与接口联调用例实现步骤。
