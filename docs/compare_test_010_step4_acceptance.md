# compare_test_010 / Step 4 验收说明

任务目标：补充“商品添加主流程测试”，覆盖 2 至 4 个不同商品加入对比页后的主区域联动更新。

## 本步骤范围

- 从商品卡片添加入口触发加入对比
- 验证对比列表按添加结果展示商品
- 验证主区域基础信息同步渲染
- 验证价格、库存或属性区域随对比数据同步更新
- 验证添加按钮状态按现有实现切换

## 建议执行命令

- `pnpm vitest run compare`
- 或 `npm run test -- compare`

## 验收清单

1. 依次添加 2 个不同商品后：
   - 对比列表展示 2 个商品
   - 商品名称、图片可见
   - 主区域同步出现对应商品列
   - 价格、库存或属性区域展示对应值
   - 已添加商品按钮切换为禁用或已添加态

2. 继续添加至 4 个不同商品后：
   - 对比列表数量与顺序和接口返回保持一致
   - 主区域列数同步增加
   - 每个商品的基础信息渲染完整
   - 已添加项按钮状态继续符合现有实现

3. 测试可重复运行：
   - 使用稳定 mock 数据
   - 不依赖真实网络
   - 每次执行结果一致

## 数据要求

测试数据至少包含：
- `id`
- `name`
- `image`
- `price`
- `stock`
- `attributes`
- `detailUrl` 或页面现有详情字段

## 风险备注

- 若仓库中商品对比页使用的按钮文案并非“加入对比/已加入”，测试应以现有实际 aria-label、data-testid 或可访问名称为准。
- 若主区域没有独立价格区块，则使用现有基础信息或属性区块断言，不新增展示规则。
